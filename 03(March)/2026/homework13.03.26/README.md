# Домашнее задание до 13.03.26
Приведите сравнительную характеристику методологии разработки ПО методом 
водопада с XP 

## Ответ
Водопадная модель (Waterfall) и Extreme Programming (XP) относятся к разным подходам к разработке программного обеспечения.
| Критерий                          | Водопадная модель (Waterfall)                                 | XP (Extreme Programming)                               |
| --------------------------------- | ------------------------------------------------------------- | ------------------------------------------------------ |
| **Подход к разработке**           | Последовательное выполнение этапов                            | Итерационная и гибкая разработка                       |
| **Планирование**                  | Детальное планирование всего проекта заранее                  | Планирование выполняется перед каждой итерацией        |
| **Требования**                    | Должны быть максимально полными и стабильными с самого начала | Могут изменяться в ходе проекта                        |
| **Участие заказчика**             | Ограничено этапами согласования                               | Постоянное взаимодействие с командой                   |
| **Документация**                  | Большой объем документации                                    | Минимально необходимая документация                    |
| **Разработка**                    | После завершения проектирования                               | Небольшими итерациями с регулярными поставками         |
| **Тестирование**                  | Обычно проводится после завершения разработки                 | Выполняется непрерывно на протяжении всего проекта     |
| **Изменения в проекте**           | Вносятся сложно и дорого                                      | Легко адаптируются                                     |
| **Скорость получения результата** | Рабочий продукт появляется в конце проекта                    | Рабочие версии поставляются регулярно                  |
| **Управление рисками**            | Риски часто обнаруживаются на поздних этапах                  | Риски выявляются и устраняются на ранних стадиях       |
| **Размер проектов**               | Подходит для небольших проектов со стабильными требованиями   | Подходит для проектов с часто меняющимися требованиями |
Основные принципы XP

### XP является одной из Agile-методологий и основана на следующих практиках:
- Частые небольшие релизы.
- Постоянное взаимодействие с заказчиком.
- Парное программирование.
- Непрерывное тестирование.
- Рефакторинг кода.
- Коллективное владение кодом.
- Простота проектных решений.
- Преимущества и недостатки
- Водопадная модель

Преимущества:
- Простота управления проектом.
- Четкое распределение этапов.
- Хорошая документированность.

Недостатки:
- Сложность внесения изменений.
- Позднее получение результата.
- Высокий риск несоответствия готового продукта новым требованиям.

### XP

Преимущества:
- Быстрая адаптация к изменениям.
- Высокое качество кода благодаря постоянному тестированию.
- Раннее получение работающего программного продукта.

Недостатки:
- Требует постоянного участия заказчика.
- Сложнее прогнозировать сроки и бюджет.
- Менее эффективна для крупных проектов с жестко фиксированными требованиями.
### Вывод
- Водопадная модель эффективна для проектов с заранее определёнными и редко изменяющимися требованиями.
- XP ориентирована на быстрое реагирование на изменения, тесное взаимодействие с заказчиком и регулярную поставку работающего программного обеспечения, что делает её более подходящей для современных динамичных проектов.
